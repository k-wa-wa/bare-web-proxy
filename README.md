# bare-web-proxy

Go と Headless Chrome（サイドカー構成）を利用して、データ通信容量を極限まで削減する超軽量な中継 Web プロキシサーバー。

## 概要

`bare-web-proxy` は、モバイル端末などの通信リソースが限られた環境向けに設計された通信量削減プロキシである。

通常のプロキシと異なり、サーバー側で対象のWebページをレンダリングした上で、**クライアントの表示やデータ節約に不要な要素を根こそぎカットした軽量 HTML** に再構築して返却する。

### 主な役割と機能
* **動的レンダリングの実行**: シングルページアプリケーション (SPA) などの JavaScript 必須なページも、サーバー側の Headless Chrome で一度レンダリングしてから HTML を取得する。
* **不要なタグの排除**: レンダリング結果から `script`, `noscript`, `iframe`, `img`, `video` などのアセットや動的処理タグをすべて削除する。
* **スタイルシート（CSS）の最適化・再適用**: 元の `<style>` タグや外部 `<link rel="stylesheet">` は一旦削除されるが、Headless Chrome で実際に適用されたスタイル（CSSテキスト）を抽出して、インライン `<style data-proxy-style="original">` として再適用する。これにより、レイアウトを極力維持しつつアセットやスクリプトを排除する。
* **リーダーモードのサポート**: 独自のリーダーモード用CSS (`reader.css`) を埋め込み、見やすさを向上させる。
* **カスタムツールバーの提供**: プロキシしたページの上部にツールバーを挿入。以下の機能を提供する。
  * オリジナルCSSの適用トグル（元のデザインとリーダーモード風の切り替え）
  * ページの再読み込み
  * オリジナルのURLを新しいタブで開く
* **ドメイン固有の修飾処理 (Modifiers)**: JavaScript 無効化によって発生する表示の不具合（例: `zenn.dev` でのスクロール不可現象）を解決するための、ドメイン固有の CSS パッチなどを適用する。
* **中継 URL への書き換え**: 取得した HTML 内のすべての `<a>` タグのリンクを自動で解析し、相対パスを絶対URLに直した上で、再度このプロキシを経由する中継 URL に書き換える。これにより、プロキシ経由でシームレスにリンク先を巡回（ブラウジング）できる。
* **通信削減率の可視化**: URL ごとに、オリジナルサイズと削減後のサイズ、および削減率を標準出力にログ出力する。

## アーキテクチャ

Kubernetes 環境において、1つの Pod 内で以下の2つのコンテナを同居させるサイドカー構成を取ることで、オーバーヘッドなしで爆速かつ省リソースな並行処理を実現している。

* **`proxy-app` (Goコンテナ)**:
  net/http で動作する軽量プロキシ。受信したリクエストを chromedp を介して隣の Chrome コンテナへ中継し、戻ってきた HTML を goquery で高速パース・軽量化してクライアントに返す。
* **`headless-chrome` (headless-shell コンテナ)**:
  `chromedp/headless-shell` イメージを使用した極限まで軽量化されたブラウザエンジン。`proxy-app` から localhost のデバッグポート経由で制御される。

## メモ

### NixOS でのファイアウォール設定

```
sudo iptables -I nixos-fw -p tcp --dport 3000 -j ACCEPT
sudo iptables -D nixos-fw -p tcp --dport 3000 -j ACCEPT
```
