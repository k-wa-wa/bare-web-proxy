# bare-web-proxy ベンチマーク環境

本ディレクトリは、`bare-web-proxy` のレスポンスタイムおよびデータ削減率を定量的に測定・検証するためのローカル性能テスト環境です。

本番コードや本番マニフェスト（`k8s/base`）に影響を与えることなく、ローカル開発環境（`k8s/overlays/local`）だけで機能するように構築されています。

---

## 構成

1.  **モックサーバー (`benchmark/mockserver/`)**
    *   性能検証のために、さまざまな特徴（アセットサイズ、遅延、動的処理など）を持つダミーページを配信するHTTPサーバー。
    *   ローカル Kubernetes 上の Pod にサイドカーコンテナとして自動デプロイされ、ポート `3003` で動作します。
2.  **測定クライアント (`benchmark/client/`)**
    *   ローカルマシンから各モックページに「直接アクセスした際」と「プロキシを経由してアクセスした際」のデータサイズおよびレスポンスタイムを複数回測定し、結果を集計して Markdown 形式の表で出力するGo製ツール。

---

## 使い方

### 1. ローカル Kubernetes へのデプロイ
以下のスクリプトを実行して、プロキシサーバーとモックサーバーをビルド・デプロイし、ポートフォワードを設定します（※すでにKindクラスター等が存在している前提で動作します）。

```bash
./benchmark/deploy.sh
```

これによって、以下のポートがローカルマシンにフォワードされます。
*   プロキシサーバー: `http://localhost:3000`
*   モックサーバー: `http://localhost:3003`

### 2. 性能測定ベンチマークの実行
ローカルマシンのターミナルから、以下のコマンドを実行します。

```bash
go run benchmark/client/main.go
```

## テストパターンの詳細

| パターン名 | パス | 特徴・検証目的 |
| :--- | :--- | :--- |
| **Simple HTML** | `/simple` | 外部CSS/JSのないプレーンなテキストHTML。プロキシ処理自体の基本オーバーヘッドを測定します。 |
| **Standard CSS** | `/css-standard` | 2枚の外部CSS（計10KB程度）を読み込む標準的なページ。外部CSSのインライン化の挙動を測定します。 |
| **Massive CSS** | `/css-massive` | 約100KBに及ぶ巨大なダミーCSSを読み込むページ。インライン化時のHTMLサイズの変化や処理性能を測定します。 |
| **Dynamic SPA** | `/spa` | ロード後、JavaScriptによって3秒間かけてDOMが動的に追加されるSPA風のページ。レンダリング待機処理の挙動を検証します。 |
| **Slow Loading CSS** | `/slow-loading` | CSSファイルの応答に意図的に2秒間の遅延が発生するページ。外部アセット読み込み待機におけるタイムアウトや耐性を検証します。 |
